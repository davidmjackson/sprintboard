# R2.2 Grounded Estimation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Each AI-proposed child work item arrives with a defensible story-point estimate that persists into the ticket's `story_points` on accept.

**Architecture:** Additively extend the existing `/decompose` flow. The model now returns a per-proposal `estimate` + `estimate_reason`; a new pure module `api/app/estimation.py` snaps each estimate onto a fixed Scrum scale and derives an epic rollup total (model maps, Python derives — deterministic and exactly testable). The React panel shows a `{n} pts` chip + reason + total, and the existing accept path passes the estimate into `createTicket({ storyPoints })`.

**Tech Stack:** FastAPI + Pydantic + pytest (`api/`), React + TypeScript + Vitest + Tailwind/shadcn (`src/`).

## Global Constraints

- **API tests:** `cd api && .venv/bin/python -m pytest -q`. **API lint:** `cd api && .venv/bin/ruff check .` (line-length 100). Both must be clean.
- **Client verify (the human runner does this at the end):** `npm run verify`. Implementers may run a focused file, e.g. `npx vitest run src/lib/ai.test.ts`. **Never** use `tsc --noEmit` as a build proxy — use `npm run build` if a type check is needed ([[verify-with-the-projects-own-scripts]]).
- **Stateless service, single mock seam:** `/decompose` verifies the JWT only and calls Claude; it touches no DB and holds no secret. `llm.propose` is the ONLY seam tests mock — CI makes no network call and needs no `ANTHROPIC_API_KEY`. Do not add DB access.
- **Structured-output schema rule:** every schema object has `additionalProperties:false` + every property in `required`; **no** `minLength`/`maxLength`/numeric (min/max) constraints. The Scrum scale is enforced in Python (`_snap`), NOT in the JSON schema.
- **Numeric rendering:** use `!= null` (never a falsy check) for point values — `0` is a real number, absence is `null` ([[sprintboard-frontend-conventions]]).
- **react-hooks/set-state-in-effect is ON and fails the build** — no synchronous `setState` at the top of a `useEffect`. This plan only sets state inside event handlers, so it's not at risk; do not introduce an effect that does.
- **Snake_case on the wire** (`estimate`, `estimate_reason`, `estimate_total`) — matches R2.0/R2.1's `covers`, `coverage_gaps`.
- **Commit** after each task with an imperative summary. Do not push; the human runner ships via `ship-story`.

- [ ] **Step 4: Write the estimation module**

Create `api/app/estimation.py`:

```python
from .schemas import DecomposeResponse

ALLOWED_POINTS: tuple[int, ...] = (1, 2, 3, 5, 8, 13)  # modified Fibonacci Scrum scale


def _snap(value: int) -> int:
    """Snap an estimate onto ALLOWED_POINTS: nearest by absolute distance, ties resolve to
    the smaller size. A model that emits 0, 7, or 100 is corrected here so the UI never
    shows an off-scale Scrum size."""
    return min(ALLOWED_POINTS, key=lambda point: (abs(point - value), point))


def analyze(response: DecomposeResponse) -> DecomposeResponse:
    """Validate every proposal's estimate onto the scale and derive estimate_total as the
    deterministic sum of the snapped estimates. Pure — the total is a function of the parts,
    never a second number from the model."""
    snapped = [p.model_copy(update={"estimate": _snap(p.estimate)}) for p in response.proposals]
    total = sum(p.estimate for p in snapped)
    return response.model_copy(update={"proposals": snapped, "estimate_total": total})
```

      proposals?: DecomposeProposal[]
      coverage_gaps?: CoverageGap[]
      scope_creep?: ScopeCreep[]
      estimate_total?: number
    }
    if (!Array.isArray(body?.proposals)) return { ok: false, error: 'request_failed' }

      ok: true,
      proposals,
      coverage_gaps: Array.isArray(body?.coverage_gaps) ? body.coverage_gaps : [],
      scope_creep: Array.isArray(body?.scope_creep) ? body.scope_creep : [],
      estimate_total: typeof body?.estimate_total === 'number' ? body.estimate_total : 0,
    }
