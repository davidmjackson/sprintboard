# S4.2 Ticket Detail and Edit Modal — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Jira-style ticket detail modal with inline click-to-edit fields, backed by an owner-scoped `updateTicket` write path.

**Architecture:** A `TicketDetailDialog` (shadcn `Dialog`) opens from a clicked `TicketCard`. Each field edits in place and commits independently via `updateTicket(id, patch)`; saves are optimistic against `ProjectShellContext` with rollback on error. `updated_at` is refreshed by the DB trigger, never sent by the client.

**Tech Stack:** React + TypeScript (strict), Vite, Tailwind, shadcn/ui, Supabase JS, Vitest (jsdom + node), react-router-dom.

## Global Constraints

- **Status, type, and column names live only in `src/lib/domain.ts`.** Never inline the four column names or type labels in a component. Use `TICKET_TYPES`, `TICKET_TYPE_LABELS`, `TICKET_STATUS_LABELS`.
- **Never send `key`, `number`, `updated_at`, `id`, or `project_id` in a ticket update.** They are trigger- or identity-owned. The `TicketUpdate` type must make them unrepresentable.
- **`updated_at` is refreshed by the `tickets_set_updated_at` BEFORE UPDATE trigger.** The app must not set it.
- **Assignee list is `{ Unassigned, current user }` only.** Do NOT widen the `profiles` SELECT policy — the `display_name` fallback is the user's email. Read no other user's profile. The current user's own id/email comes from `useAuth()`.
- **Native `<select>`, not radix Select** (jsdom-testable, no extra dependency), styled with the shared `selectClass` pattern already in `CreateTicketDialog.tsx`.
- **No refetch on edit.** Update the ticket in `ProjectShellContext` state in place, mirroring the S4.1 optimistic-append pattern (a refetch race was a MAJOR finding there).
- **Verification is `npm run verify`.** Not a subset. CI runs the same command.
- Scope is exactly the seven AC fields + inline editing + the two-column layout. Status-change, block/unblock (S4.4), and epic context/deliverables (S4.5) are out of scope and shown read-only or omitted.

```ts
/**
 * Update a ticket's editable fields. `key`, `number`, `id`, `project_id` and the
 * timestamps are excluded by `TicketUpdate`, so the immutable and trigger-owned columns
 * cannot be sent. `updated_at` is refreshed by the `tickets_set_updated_at` trigger, so
 * the returned row carries the new timestamp. RLS (`tickets_owner`) scopes the write
 * through the owned project: a cross-tenant update matches zero rows, `.single()` then
 * errors, and we report the single non-user-correctable `'unknown'`.
 */
export type UpdateTicketResult = { ok: true; ticket: Ticket } | { ok: false; error: 'unknown' }

export async function updateTicket(id: string, patch: TicketUpdate): Promise<UpdateTicketResult> {
  const { data, error } = await supabase
    .from('tickets')
    .update(patch)
    .eq('id', id)
    .select()
    .single()

  if (error) return { ok: false, error: 'unknown' }
  return { ok: true, ticket: data as Ticket }
}
```

```ts
/** "ui, urgent ," -> ["ui", "urgent"]. Trims each label, drops blanks. */
export function parseLabels(raw: string | undefined): string[] {
  return (raw ?? '')
    .split(',')
    .map((label) => label.trim())
    .filter(Boolean)
```

```tsx
import { useEffect, useState } from 'react'

import { updateTicket } from '@/lib/tickets'
import { parseLabels } from '@/lib/labels'
import {
  TICKET_TYPES,
  TICKET_TYPE_LABELS,
  TICKET_STATUS_LABELS,
  type Ticket,
  type TicketType,
  type TicketUpdate,
} from '@/lib/domain'
import { Input } from '@/components/ui/input'
import { Textarea } from '@/components/ui/textarea'
import {
  Dialog,
  DialogContent,
  DialogHeader,
  DialogTitle,
} from '@/components/ui/dialog'

const selectClass =
  'border-input focus-visible:border-ring focus-visible:ring-ring/50 h-8 w-full rounded-lg border bg-transparent px-2.5 py-1 text-base transition-colors outline-none focus-visible:ring-3 md:text-sm'

/** Click-to-edit text/number field. View mode is a button; edit mode is an input or
 *  textarea. Enter (single-line) or blur commits a changed value; Esc cancels. */
function EditableText({
  value, ariaLabel, multiline, numeric, placeholder, onCommit,
}: {
  value: string
  ariaLabel: string
  multiline?: boolean
  numeric?: boolean
  placeholder?: string
  onCommit: (next: string) => void
}) {
  const [editing, setEditing] = useState(false)
  const [draft, setDraft] = useState(value)

  function start() {
    setDraft(value)
    setEditing(true)
  }
  function commit() {
    setEditing(false)
    if (draft !== value) onCommit(draft)
  }
  function cancel() {
    setEditing(false)
    setDraft(value)
  }

  if (!editing) {
    return (
      <button
        type="button"
        aria-label={`Edit ${ariaLabel}`}
        onClick={start}
        className="hover:bg-muted/60 w-full rounded px-2 py-1 text-left text-sm"
      >
        {value || <span className="text-muted-foreground">{placeholder ?? 'Empty'}</span>}
      </button>
    )
  }

  const commonProps = {
    autoFocus: true,
    'aria-label': ariaLabel,
    value: draft,
    onChange: (e: { target: { value: string } }) => setDraft(e.target.value),
    onBlur: commit,
    onKeyDown: (e: React.KeyboardEvent) => {
      if (e.key === 'Escape') cancel()
      if (e.key === 'Enter' && !multiline) {
        e.preventDefault()
        commit()
      }
    },
  }

  return multiline ? (
    <Textarea rows={3} {...commonProps} />
  ) : (
    <Input type={numeric ? 'number' : 'text'} min={numeric ? 0 : undefined} placeholder={placeholder} {...commonProps} />
  )
}

/**
 * Jira-style ticket detail modal. Every field edits in place and commits independently.
 * Saves are optimistic against `onUpdated`, persisted with `updateTicket`, and rolled
 * back on failure. `updated_at` comes from the DB trigger via the reconciled row.
 * Assignee is deliberately `{ Unassigned, current user }` — Phase 1 is single-owner, and
 * widening the profiles read would leak every user's email.
 */
export function TicketDetailDialog({
  ticket,
  currentUser,
  onOpenChange,
  onUpdated,
}: {
  ticket: Ticket | null
  currentUser: { id: string; email: string }
  onOpenChange: (open: boolean) => void
  onUpdated: (ticket: Ticket) => void
}) {
  const [error, setError] = useState<string | null>(null)
  // Clear a stale error whenever a different ticket opens.
  useEffect(() => setError(null), [ticket?.id])

  if (!ticket) return null

  async function commit(patch: TicketUpdate) {
    const prev = ticket!
    onUpdated({ ...prev, ...patch } as Ticket) // optimistic — instant
    const result = await updateTicket(prev.id, patch)
    if (!result.ok) {
      onUpdated(prev) // rollback
      setError('Could not save your change. Please try again.')
    } else {
      onUpdated(result.ticket) // reconcile DB-refreshed updated_at
      setError(null)
    }
  }

  const assigneeValue = ticket.assignee_id === currentUser.id ? currentUser.id : ''

  return (
    <Dialog open={ticket !== null} onOpenChange={onOpenChange}>
      <DialogContent className="sm:max-w-3xl">
        <DialogHeader>
          <DialogTitle className="flex items-center gap-2 text-base">
            <span className="text-muted-foreground font-mono text-sm">{ticket.key}</span>
            <span className="bg-muted text-muted-foreground rounded px-1.5 py-0.5 text-[10px] font-medium uppercase">
              {TICKET_TYPE_LABELS[ticket.type]}
            </span>
            <span className="text-muted-foreground text-xs">{TICKET_STATUS_LABELS[ticket.status]}</span>
          </DialogTitle>
        </DialogHeader>

        <div className="grid gap-6 sm:grid-cols-[1fr_220px]">
          {/* Main column: summary + description */}
          <div className="flex flex-col gap-4">
            <div>
              <h3 className="text-muted-foreground mb-1 text-xs font-medium uppercase">Summary</h3>
              <EditableText value={ticket.summary} ariaLabel="summary" onCommit={(v) => commit({ summary: v })} />
            </div>
            <div>
              <h3 className="text-muted-foreground mb-1 text-xs font-medium uppercase">Description</h3>
              <EditableText
                value={ticket.description ?? ''}
                ariaLabel="description"
                multiline
                placeholder="Add a description…"
                onCommit={(v) => commit({ description: v.trim() || null })}
              />
            </div>
            <div>
              <h3 className="text-muted-foreground mb-1 text-xs font-medium uppercase">Acceptance criteria</h3>
              <EditableText
                value={ticket.acceptance_criteria ?? ''}
                ariaLabel="acceptance criteria"
                multiline
                placeholder="Add acceptance criteria…"
                onCommit={(v) => commit({ acceptance_criteria: v.trim() || null })}
              />
            </div>
          </div>

          {/* Sidebar: type, assignee, points, labels */}
          <aside className="flex flex-col gap-4 border-t pt-4 sm:border-t-0 sm:border-l sm:pt-0 sm:pl-4">
            <label className="flex flex-col gap-1">
              <span className="text-muted-foreground text-xs font-medium uppercase">Type</span>
              <select
                aria-label="type"
                className={selectClass}
                value={ticket.type}
                onChange={(e) => commit({ type: e.target.value as TicketType })}
              >
                {TICKET_TYPES.map((t) => (
                  <option key={t} value={t}>{TICKET_TYPE_LABELS[t]}</option>
                ))}
              </select>
            </label>

            <label className="flex flex-col gap-1">
              <span className="text-muted-foreground text-xs font-medium uppercase">Assignee</span>
              <select
                aria-label="assignee"
                className={selectClass}
                value={assigneeValue}
                onChange={(e) => commit({ assignee_id: e.target.value || null })}
              >
                <option value="">Unassigned</option>
                <option value={currentUser.id}>{currentUser.email}</option>
              </select>
            </label>

            <div className="flex flex-col gap-1">
              <span className="text-muted-foreground text-xs font-medium uppercase">Story points</span>
              <EditableText
                value={ticket.story_points?.toString() ?? ''}
                ariaLabel="story points"
                numeric
                placeholder="—"
                onCommit={(v) => commit({ story_points: v.trim() === '' ? null : Number(v) })}
              />
            </div>

            <div className="flex flex-col gap-1">
              <span className="text-muted-foreground text-xs font-medium uppercase">Labels</span>
              <EditableText
                value={ticket.labels.join(', ')}
                ariaLabel="labels"
                placeholder="Add labels…"
                onCommit={(v) => commit({ labels: parseLabels(v) })}
              />
            </div>

            <p className="text-muted-foreground text-[11px]">
              Updated {new Date(ticket.updated_at).toLocaleString()}
            </p>
          </aside>
        </div>

        {error ? (
          <p role="alert" className="text-destructive text-sm">{error}</p>
        ) : null}
      </DialogContent>
    </Dialog>
  )
}
```
