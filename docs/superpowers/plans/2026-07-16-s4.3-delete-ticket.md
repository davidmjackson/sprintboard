# S4.3 Delete Ticket ÔÇö Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a user delete a single ticket from its detail modal, behind a Jira-style kebab Actions menu and a confirm step, so it disappears from the board and backlog.

**Architecture:** Add a `deleteTicket` write to the data layer, two shadcn primitives (`DropdownMenu`, `AlertDialog`) from the already-installed `radix-ui` package, a kebab Actions menu + confirm inside the existing `TicketDetailDialog`, and an `onTicketDeleted` handler on `ProjectShellContext` that removes the row from shared state (no refetch). RLS already covers DELETE (`tickets_owner FOR ALL`); a live cross-tenant test proves isolation.

**Tech Stack:** React + Vite + TypeScript (strict), Tailwind, shadcn/ui over the unified `radix-ui` package, Supabase (Postgres + RLS), Vitest + Testing Library (jsdom), Playwright (not used here).

## Global Constraints

- **Delete a TICKET only** ÔÇö never the board (fixed four columns, not an entity) or a project (no delete-project story exists). One row in `tickets`.
- **A confirm step is mandatory** ÔÇö a bare menu click must never delete (AC1).
- **Delete is NOT optimistic.** `await deleteTicket`; remove + close only on success; on failure keep the ticket and show the error banner.
- **Zero rows returned = failure**, not silent success. `deleteTicket` uses `.select()` and treats an empty result set as `{ ok: false }` (RLS filters to zero rows; it does not raise).
- **No refetch.** Removal is a local context mutation (`onTicketDeleted(id)`), guarded by `prev.projectId === project.id` ÔÇö mirrors S4.1/S4.2.
- **No new dependencies.** `DropdownMenu`/`AlertDialog` come from the unified `radix-ui` package already installed (`import { DropdownMenu as DropdownMenuPrimitive } from 'radix-ui'`). No `@radix-ui/react-*` per-package imports.
- **Status/type/column display names come only from `src/lib/domain.ts`** ÔÇö never inline them.
- **RLS is owner-scoped and unchanged.** `tickets_owner` is `FOR ALL`, so it already covers DELETE. Do not add or alter any policy.
- **Keys are never reused.** Delete must not touch `project_counters`; a deleted `MP-5` leaves a gap and `MP-6` is still next.
- Mutations in `src/lib/tickets.ts` return a discriminated union `{ ok: true; ÔÇŽ } | { ok: false; error: ÔÇŽ }`; a throw is for unexpected errors only.
- Scope: delete only. No Clone/Move/Archive/Export menu items, no block/unblock (S4.4), no delete-from-card, no undo/soft-delete.

```ts
/**
 * Delete a ticket by id. RLS (`tickets_owner`, `FOR ALL`) scopes the delete through the
 * owned project, so a cross-tenant delete matches ZERO rows rather than raising. We
 * `.select()` the deleted rows and treat an empty result set as a failure ÔÇö a delete that
 * removed nothing is not a success. Deleting a parent epic nulls its children's
 * `parent_epic_id` (the `tickets_epic_fk` `on delete set null`), and `project_counters` is
 * untouched, so ticket numbers are never reused. A failure is not user-correctable here.
 */
export type DeleteTicketResult = { ok: true } | { ok: false; error: 'unknown' }

export async function deleteTicket(id: string): Promise<DeleteTicketResult> {
  const { data, error } = await supabase.from('tickets').delete().eq('id', id).select()

  if (error) return { ok: false, error: 'unknown' }
  if (!data || data.length === 0) return { ok: false, error: 'unknown' }
  return { ok: true }
}
```

```ts
async function handleDelete() {
  const id = ticket!.id
  setDeleting(true)
  const result = await deleteTicket(id)
  if (!mountedRef.current) return // dialog was dismissed while the delete was in flight
  if (result.ok) {
    // Parent removes the row Ôćĺ `ticket` becomes null Ôćĺ this dialog unmounts. We don't
    // reset local state (we're on our way out) and never close ourselves directly.
    onDeleted(id)
  } else {
    setDeleting(false)
    setConfirmingDelete(false)
    setErrorFor({ ticketId: id, message: 'Could not delete this ticket. Please try again.' })
  }
}
```
