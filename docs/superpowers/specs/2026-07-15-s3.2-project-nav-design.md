# S3.2 — Left-nav project list and switcher (design)

**Story (E3 Projects):** the current user's projects list in the left nav; selecting a
project loads it and the choice survives a refresh; another user's projects never
appear (RLS).

## Approach

- **Selection lives in the URL.** Selecting a project navigates to `/projects/:projectId`;
  a refresh restores it from the route. No client-side "current project" store to keep in
  sync — the URL *is* the state, and it's where S3.3's project shell will render.
- **An `AppLayout` shell** (left sidebar + `<Outlet>`) replaces the flat placeholder. The
  sidebar holds the RLS-scoped project list, the create-project dialog, and the user /
  logout controls. The layout fetches once and hands `{projects, loading}` to its routed
  children via `Outlet` context.
- **RLS is the isolation.** `listProjects()` is a plain `select` — the `projects_owner`
  policy scopes it to `owner_id = auth.uid()`, so another owner's projects are never
  returned. Proven live.

## Components

| File | Purpose |
|---|---|
| `src/lib/projects.ts` | add `listProjects()` — name-ordered, RLS-scoped select. |
| `src/routes/AppLayout.tsx` | the authed shell: sidebar (project nav via `NavLink`, create dialog wired to refetch + navigate, user/logout) + `<Outlet context>`. |
| `src/routes/ProjectsHome.tsx` | `/` landing: empty / "select a project" state. |
| `src/routes/ProjectView.tsx` | `/projects/:projectId`: looks the project up in the RLS-scoped list; an id not in the list → `<Navigate to="/">` (never reveal another owner's). Placeholder until S3.3. |
| `src/App.tsx` | routes restructured: guard → `AppLayout` → `{ /, /projects/:id, * }`. |

## Testing

- `listProjects` unit-tested (success / empty / error).
- **Live isolation test** (`projects.integration.test.ts`): A and B each create a project;
  A's `select` contains A's id and excludes B's — the AC's RLS requirement, proven with two
  real owners.
- `AppLayout.test.tsx`: list renders; empty state; **selecting opens the project**;
  **the project is restored from the URL on load (refresh survival)**; an unknown id sends
  the user home.
- `App.test.tsx` updated for the new shell (the persisted-session/logout tests now assert
  the shell's controls instead of the removed placeholder text).

## Out of scope

The project view is a placeholder — **Board and Backlog tabs are S3.3**. No project
rename/delete (later). Auto-selecting a project on load is deliberately not done; `/` shows
a neutral prompt (the AC asks only that *selecting* loads and survives refresh).
